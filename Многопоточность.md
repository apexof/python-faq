**Многопоточность (multithreading)** в Python — выполнение нескольких потоков (threads) в рамках одного процесса с помощью модуля `threading`. Потоки делят общую память процесса, поэтому могут напрямую обращаться к одним и тем же объектам без межпроцессного обмена данными.

## Ограничение: GIL

В [[Реализации интерпретатора Python|CPython]] потоки ограничены [[GIL]] — в один момент времени байт-код Python исполняет только один поток. Из-за этого:

- **I/O-bound** задачи (сеть, диск, ожидание) — `threading` даёт реальный выигрыш, так как GIL освобождается на время ожидания I/O.
- **CPU-bound** задачи (вычисления на чистом Python) — параллелизма нет, потоки исполняются по очереди.

Подробный механизм — в [[GIL]].

## Основные объекты модуля `threading`

| Объект | Назначение |
|---|---|
| `Thread` | Класс для создания и запуска потока |
| `Lock` | Простая блокировка — взаимоисключающий доступ к ресурсу |
| `RLock` | Реентерабельная блокировка — тот же поток может захватить её повторно без дедлока |
| `Event` | Флаг для сигнализации между потоками (`set()`, `wait()`, `clear()`) |
| `Condition` | Блокировка + возможность ждать определённого условия |
| `Semaphore` | Ограничивает число потоков, одновременно имеющих доступ к ресурсу |
| `local` (`threading.local`) | Хранилище данных, уникальное для каждого потока |

## Создание потока

```python
import threading

def worker(n):
    print(f"поток {n} работает")

threads = []
for i in range(3):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()  # ждём завершения каждого потока
```

- `start()` — запускает поток.
- `join()` — блокирует текущий поток до завершения целевого.
- Поток также можно создать через наследование от `Thread` и переопределение метода `run()`.

## Гонка данных и синхронизация

Несмотря на GIL, гонки данных (race condition) возможны: GIL защищает только внутренние структуры интерпретатора (например, счётчик ссылок), но не логику программы — операция `x += 1` состоит из нескольких байт-код инструкций, между которыми GIL может переключиться на другой поток.

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    for _ in range(100_000):
        with lock:   # ✅ без lock возможна гонка данных
            counter += 1

threads = [threading.Thread(target=increment) for _ in range(2)]
for t in threads:
    t.start()
for t in threads:
    t.join()

print(counter)  # 200000 — гарантировано только благодаря lock
```

## Daemon-потоки

Поток, созданный с `daemon=True`, автоматически завершается при завершении главной программы, не дожидаясь своего конца — в отличие от обычных потоков, которые держат процесс живым, пока не завершатся сами.

```python
t = threading.Thread(target=worker, args=(1,), daemon=True)
```

## Когда использовать `threading`

- I/O-bound нагрузка: сетевые запросы, работа с файлами, ожидание ответов от БД.
- Не подходит для ускорения CPU-bound вычислений на чистом Python — для этого нужен [[Мультипроцессинг]].
