#### Как устроена память в питоне?

**1. Все - объекты**
Переменные хранят ссылки на объекты, а не сами значения.
Объект содержит: значение, тип, счётчик ссылок.

**2. Reference Counting**
Каждый объект имеет счётчик ссылок `ob_refcnt`.
Когда счётчик = 0, память освобождается.

```python
import sys
sys.getrefcount(a)  # показывает количество ссылок
```
#### Что такое garbage collector?
Garbage collector (GC) — автоматическая система удаления объектов, на которые больше никто не ссылается. Находит и удаляет циклические ссылки (A → B → A). Работает периодически или при превышении порогов.
#### Интернирование
Интернирование - переиспользование объектов в памяти для разных переменных.
Объекты в памяти создаются заранее, при создании переменной новый объект не создается, ссылка будет указывать на уже созданный объект.

Что интернируется:
- Числа: -5 … 256
- Строки: литералы-идентификаторы (буквы/цифры/`_`), известные при компиляции
- bool — True, False: всегда один объект
- None: единственный экземпляр в программе, поэтому проверяют `x is None`.
- пустой tuple

#### Free lists (Свободные списки)
Free lists - пулы переиспользуемой памяти для часто создаваемых объектов (tuple, list, dict, float, int, [[Стек вызовов|frames]]…)
Когда объект удаляется (refcount → 0), CPython не возвращает его память ОС, а кладёт «пустой слот» в список — free list. При следующем создании объекта того же типа берётся готовый слот из пула, и в него записывается новое содержимое.

#### Constant folding
Constant folding — оптимизация на этапе компиляции: вычисление выражений из констант заранее, чтобы не считать их каждый раз в рантайме. Результат сразу кладётся в байткод.

```python
x = 2 + 3        # компилятор сразу пишет x = 5
s = "ab" * 3     # → "ababab"
```

В байткоде уже лежит готовая константа, а не инструкции сложения.

**Что сворачивается:**
- арифметика над литералами: `2 + 3`, `10 ** 2`
- операции над строками/байтами: `"a" + "b"`, `"x" * 5`
- кортежи из констант: `(1, 2, 3)`
- булева/сравнения констант в некоторых случаях

**Что НЕ сворачивается:**
- переменные: `x + 3` — значение `x` неизвестно при компиляции
- слишком большие результаты: `2 ** 100000` — иначе раздует байткод и замедлит старт (есть лимит)
- вызовы функций: `len("abc")` — функция могла быть переопределена

#### Алокатор
**Аллокатор (allocator)** — это компонент, который управляет выделением и освобождением памяти: по запросу выдаёт блок нужного размера, а при освобождении забирает обратно для повторного использования.

В Python объекты создаются постоянно и в основном мелкие (int, маленькие строки, узлы структур). Звать системный `malloc`/`free` напрямую на каждый объект — дорого. Поэтому CPython ставит свой аллокатор поверх системного.
