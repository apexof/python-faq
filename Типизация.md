#### Какие типы изменяемые а какие нет?
Изменяемые: list, dict, set, bytearray (еще класс, функция, модуль, инстанс класса)
Неизменяемые: int, float, str, bool, NoneType, tuple, frozenset, frozendict

#### Какая типизация в Python?
Динамическая — тип переменной определяется во время выполнения:

```python
x = 5       # int
x = "text"  # str - переприсвоение без ошибки
```

Строгая — Python не выполняет неявных приведений между несовместимыми типами.
`"5" + 5` бросит `TypeError`.

#### Какие виды приведения типов есть в Python?

##### Неявное — интерпретатор сам приводит типы в безопасных случаях. В основном это касается числовых операций:

```python
x = 5 + 2.0   # int + float -> float, x = 7.0
y = True + 1  # bool -> int, y = 2 (bool это подкласс int)
```

##### Явное (type casting) — ты сам вызываешь функцию-конструктор типа:

```python
int("42")        # 42
int(3.9)         # 3 (отбрасывает дробную часть, не округляет)
float("3.14")    # 3.14
str(42)          # "42"
list("abc")      # ['a', 'b', 'c']
tuple([1, 2])    # (1, 2)
set([1, 1, 2])   # {1, 2}
dict([("a", 1)]) # {"a": 1}
bool(0)          # False
```

Несколько нюансов:
- `int()` от строки требует валидное целое — `int("3.14")` бросит `ValueError`
- Эти функции не «меняют» объект, а создают новый.
- Для пользовательских классов поведение приведения задаётся дандер-методами: `__int__`, `__float__`, `__str__`, `__bool__`

#### Какие значения считаются ложными (falsy)?

```python
None
False
0    # int
0.0  # float
""   # пустая строка
[]   # пустой список
()   # пустой кортеж
{}   # пустой словарь
set()  # пустое множество
frozenset()     # пустое неизменяемое множество
```

Под капотом truthiness определяется дандер-методами: сначала вызывается `__bool__()`, а если его нет — `__len__()` (объект ложный, если длина 0).
