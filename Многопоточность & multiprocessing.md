#### Что такое I/O-bound задача?
**I/O-bound задача** — задача, ограниченная операциями ввода-вывода.
Примеры задач: сетевые запросы, работа с БД, работа с файлами.

#### Что такое CPU-bound задача?
**CPU-bound задача** — задача, ограниченная мощностью процессора. Её можно ускорить, если взять более мощный процессор или задействовать больше ядер (для тех задач, которые можно распараллелить).
Примеры задач: обучение моделей, хеширование, шифрование, сжатие данных, парсинг больших объёмов, перебор/брутфорс, обработка изображений и видео.

#### Что такое multiprocessing?
**Multiprocessing** — выполнение задачи в нескольких параллельных процессах.

- Реализован модулем `multiprocessing`.
- Каждый процесс имеет свой экземпляр интерпретатора, GIL и пространство памяти.
- Данные между процессами передаются через сериализацию.
- Подходит для CPU-bound задач: тяжёлые вычисления можно распараллелить по процессам.

#### Что такое многопоточность?
**Многопоточность (threading)** — выполнение задачи в нескольких потоках внутри одного процесса.

- Реализована модулем `threading`.
- На все потоки один экземпляр интерпретатора и пространство памяти.
- Из-за GIL одновременно выполняется байткод только в одном потоке.
- Подходит для I/O-bound задач: пока один поток ждёт, другой работает.

#### Что такое GIL?
**GIL (Global Interpreter Lock)** — глобальная блокировка интерпретатора, которая позволяет выполнять байткод только одному потоку в каждый момент времени.

- Делает операции со счётчиками ссылок последовательными, без race condition.
- Во время ожидания поток отпускает GIL, и другой поток начинает работать.
- Это деталь реализации CPython, в других интерпретаторах может не быть GIL.

#### В чём отличие многопоточности от multiprocessing?
- Потоки делят память между собой, процессы — нет (передают данные через сериализацию).
- Многопоточность подходит для I/O-bound задач, multiprocessing — для CPU-bound задач.
- Многопоточность ограничена GIL, multiprocessing — нет.
