#### I/O-bound задачи
I/O-bound задача — задача, ограниченная операциями ввода-вывода:  сеть, БД, работа с файлами.
#### CPU-bound задачи
CPU-bound задача - задача ограниченная мощностью процессора. Ее можно ускорить если взять более мощный процессор или задействовать больше ядер (для тех задач которые можно распараллелить).
Примеры зада: обучение моделей, хеширование, шифрование, сжатие данных, парсинг больших объёмов, перебор/брутфорс, обработка изображений и видео.
#### Multiprocessing
Multiprocessing - параллельное выполнение кода в нескольких процессах.
- Реализован модулем `multiprocessing`.
- Каждый процесс имеет свой экземпляр интерпретатора, GIL и пространство памяти.
- Данные между процессами передаются через сериализацию 
- Подходит для CPU-bound задач, тяжелые вычисления можно распараллелить по процессам

#### Многопоточность
**Многопоточность (threading)** — запуск нескольких потоков внутри одного процесса.

- Реализована модулем `threading`.
- На все потоки один экземпляр интерпретатора и пространство памяти
- Из-за GIL одновременно выполняется байткод только в одном потоке
- Подходит для I/O-bound задач, пока один поток ждет, другой работает.

#### Что такое GIL?
**GIL (Global Interpreter Lock)** — глобальная блокировка интерпретатора, которая позволяет выполнять байткод только одному потоку в каждый момент времени.

- Это деталь реализации CPython, в других интерпретаторах может не быть GIL
- Делает операции со счётчиками ссылок последовательными, без race condition
- Во время ожидания поток отпускает GIL и другой поток начинает работать

#### Отличие многопоточность от Multiprocessing'а?
- потоки делят память между собой, процессы нет (передают данные через сериализацию)
- многопоточность подходит для I/O-bound задач, multiprocessing для CPU-bound задач
- многопоточность ограничена GIL, multiprocessing - нет
