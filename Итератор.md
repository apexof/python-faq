#### Итератор?
Итератор - это объект который можно перебирать.
Выдает элементы по одному при вызове next(iter) в конце бросает StopIteration.
Помнит позицию на которой находится перебор.
Одноразовый - его можно перебрать только 1 раз.
Чтобы реализовать класс итератора необходимо описать 2 метода

- `__iter__()`: Возвращает сам объект-итератор.
- `__next__()`: Возвращает следующее значение или выбрасывает `StopIteration`

```python
class Counter:
    def __init__(self, high):
        self.current = 0
        self.high = high

    def __iter__(self):
        return self  # Итератор возвращает сам себя

    def __next__(self):
        if self.current > self.high:
            raise StopIteration
        else:
            value = self.current
            self.current += 1
            return value
```

#### Зачем нужен итератор?
- **Дать единый способ перебора** — `for` работает одинаково с list, dict, set, файлом и любым своим объектом.
- **Перебирать лениво** — по одному элементу, не загружая всё в память.
- **Скрыть внутреннее устройство** — потребитель просто берёт `next()`, не зная, как объект хранит данные.


#### Что такое Iterable?
Iterable - это фабрика итераторов. По запросу iter() выдает свежий итератор.

```python
my_list = [1, 2, 3] # Iterable
it = iter([1, 2, 3]) # получаем свежий итератор it
print(next(it))  # получаем элементы по одному
```

Iterable важны так как итераторы одноразовые и без Iterable нельзя перебрать одни и те же данные дважды.

Чтобы создать Iterable:

```python
class CounterIterable:
    def __init__(self, limit):
        self.limit = limit

    def __iter__(self):
        return Counter(self.limit)
```

#### Что вызывает for in?
Перебор for in и для Iterable и для итератора вызывает метод iter (у обоих он есть).
Итератор вернет себя, а Iterable вернет свежий итератор.
For in вызывает next на каждой итерации.

#### Генератор?
**Генератор** — компактный способ создать итератор: пишешь функцию с `yield` вместо целого класса с `__iter__`/`__next__`.

```python
def my_gen():
    yield 1
    yield 2

for x in my_gen():
    print(x)
```

#### Зачем ещё нужен генератор, кроме создания итератора?
- **Корутин** — через `.send()` принимает значения внутрь, становясь двусторонним каналом (основа асинхронности до `async/await`).
- **Управления через `.throw()` и `.close()`** — пробросить исключение внутрь генератора или принудительно остановить его, перехватив `GeneratorExit` для очистки.
- **Ленивых пайплайнов** — генераторы цепляются в конвейер, данные текут по одному элементу за один проход.
- **Управления ресурсами** — `try/finally` с `yield` освобождает ресурс при завершении (основа `@contextmanager`).

#### Как создать генератор?
**1. Функция-генератор** — `def` с `yield` внутри:

```python
def gen():
    yield 1
    yield 2
```

**2. Generator expression** — генераторное выражение, вообще без функции и без `yield`:

```python
g = (x*x for x in range(10))   # это уже генератор
type(g)                         # <class 'generator'>
```
