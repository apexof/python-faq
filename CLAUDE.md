# Python Конспекты — CLAUDE.md

Это хранилище конспектов по Python-разработке, написанных в Obsidian.

## Структура папок

Темы разбиты по технологиям и областям знаний:

```
Python/
├── Основы/              # Переменные, типы данных, операторы, циклы
├── Функции/             # Функции, аргументы, замыкания, декораторы
├── ООП/                 # Классы, наследование, магические методы
├── Структуры данных/     # Списки, словари, множества, кортежи
│   └── Comprehensions/   # Подтемы — отдельная папка
├── Асинхронность/        # asyncio, корутины, event loop
├── Стандартная библиотека/ # Модули stdlib: os, itertools, collections...
├── Тестирование/          # pytest, unittest, моки
├── Окружение/             # venv, pip, poetry, управление зависимостями
└── DevTeam/               # Дополнительные конспекты (черновики / старые)
```

**Принцип:** одна тема — один файл. Если подтем много, создаётся папка с файлом-оглавлением и отдельными файлами для каждой подтемы.

## Стиль написания

### Язык
- Весь текст **на русском**, термины и код — на английском.
- Смешивать языки в одном предложении — нормально: "**Comprehension** — это синтаксис Python для...".

### Структура файла
- Первая строка — **определение** жирным шрифтом или обычным текстом: что это такое, одной фразой.
- Далее — разделы `##` с конкретными аспектами темы.
- Разделы короткие и конкретные, без лишней воды.

### Форматирование
- Активно используются таблицы для сравнений и перечней.
- Примеры кода — с комментариями `# ✅ правильно` / `# ❌ неправильно` прямо в коде.
- Эмодзи ✅ ❌ ⚠️ для пометок — это норма и приветствуется.
- Заголовки с нумерацией — когда важен порядок. Уровень (`##` или `####`) выбирается по визуальной компактности в Obsidian: `####` даёт меньше отступов и подходит для коротких конспектов.

### Ссылки (Obsidian wikilinks)
- Ссылки на связанные файлы через `[[ИмяФайла]]`.
- Если текст ссылки отличается от имени файла: `[[текст|ИмяФайла]]`.
- Ссылки ставятся inline в тексте при первом упоминании термина.
- **Не добавлять** разделы "Смотрите также" или "Связанные разделы" в конце файла.

### Примеры кода
- Всегда указывать язык в блоке кода: ` ```python`, ` ```shell`.
- Примеры — рабочие и минимальные, без лишнего обвеса.
- Показывать и правильный, и неправильный варианты когда это уместно.

## Чего не делать

- Не добавлять раздел "Смотрите также" / "Связанные разделы".
- Не писать длинные введения — сразу к сути.
- Не дублировать информацию из других файлов — ставь ссылку.
- Не создавать файлы вне тематических папок (исключение: файлы на корневом уровне типа `Регулярные выражения.md` — для тем без подпапки).
- **Не давать краткую таблицу/список, а потом расширять то же самое ниже** — это дублирование. Выбери один формат и придерживайся его.

## Эталон стиля

Компактно, по делу, без воды, без повторов. Каждый новый файл должен быть таким же.
