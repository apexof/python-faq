#### Что такое стек вызовов?
Стек вызовов (call stack) — структура, в которой интерпретатор хранит цепочку активных вызовов функций. Работает по принципу LIFO: при вызове функции наверх кладётся новый кадр (frame), при `return` — снимается.

```python
def a():
    b()      # 2. поверх a кладётся кадр b
def b():
    c()      # 3. поверх b кладётся кадр c
def c():
    ...      # стек снизу вверх: a -> b -> c
a()          # 1. кладётся кадр a
```

#### Что такое frame (кадр стека)?
Frame — объект, который создаётся на каждый вызов функции и хранит её контекст выполнения:
- [[Переменная|локальные переменные]] (`f_locals`)
- ссылку на исполняемый код (`f_code`)
- текущую позицию в байткоде
- ссылку на кадр вызывающей функции (`f_back`)

Поэтому после `return` локальные переменные функции исчезают — вместе с её кадром снимается весь контекст.

#### Что такое рекурсия с точки зрения стека?
Каждый рекурсивный вызов кладёт на стек новый кадр и не снимает его, пока вложенный вызов не вернётся. Глубокая рекурсия растит стек линейно по глубине.

#### Зачем нужен лимит рекурсии?
Стек не бесконечный. CPython ограничивает глубину (по умолчанию 1000), чтобы переполнение питоновского стека не уронило весь процесс по переполнению нативного C-стека. При превышении — `RecursionError`.

```python
import sys
sys.getrecursionlimit()      # 1000
sys.setrecursionlimit(3000)  # можно поднять, но осторожно

def rec():
    return rec()
rec()  # RecursionError: maximum recursion depth exceeded
```

#### Что такое traceback?
Traceback — это снимок стека вызовов в момент исключения. Питон печатает кадры от внешнего вызова к месту ошибки, поэтому самая нижняя строка — где исключение возникло.

```python
def a(): b()
def b(): c()
def c(): raise ValueError("boom")
a()
```

```
Traceback (most recent call last):
  File ..., line 4, in <module>
  File ..., line 1, in a
  File ..., line 2, in b
  File ..., line 3, in c
ValueError: boom
```
