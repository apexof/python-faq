#### В чем разница между == и is?
`==` - Вызывает метод `__eq__`.
У встроенных типов он сравнивает по значению.
Сначала вызывает `a.__eq__(b)`, если его нет то `b.__eq__(a)`.
Если метод `__eq__` не реализован он берется из `object` так как все классы от него наследуются.
В `object` метод `__eq__` реализован как a is b.

`is` - проверяет, ссылаются ли две переменные на один и тот же объект.
По сути эквивалентно `id(x) == id(y)`, но на практике `is` работает напрямую и быстрее.

Проверяет являются ли две переменные одним и тем же объектом в памяти.

```python
# Списки с одинаковым содержимым
a = [1, 2, 3]
b = [1, 2, 3]

a == b  # True - значения равны
a is b  # False - разные объекты в памяти

# Одна ссылка
c = a
c is a  # True - один объект
c == a  # True - значения тоже равны
```

#### Что вернёт id()?
`id()` возвращает целое число — уникальный идентификатор объекта, постоянный на протяжении жизни этого объекта.

```python
a = []
id(a)  # напр. 140234567891234
```

Уникальность только среди живущих одновременно объектов. После уничтожения объекта его id может быть переиспользован новым объектом. Классическая ловушка:

```python
id([1,2,3]) == id([4,5,6])  # может вернуть True!
# первый список умирает сразу, второй занимает тот же адрес
```
