#### Что такое список list?
- Изменяемая коллекция упорядоченных элементов.
- Элементы упорядочены, порядок добавления сохраняется.
- Хранит указатели на объекты, а не сами значения.
- Изменяемая.
- значения не уникальны
- Элементы могут быть разных типов.
- Элементы лежат подряд, без пропусков, дырок быть не может.

#### Алгоритмическая сложность операций списка?
- Доступ по индексу (`lst[i]`, `lst[i] = x`, `len`) — O(1).
- Операции с концом (`append`, `pop()`) — O(1) в среднем.
- Всё, что трогает начало/середину (`insert(0, x)`, `pop(0)`, `del lst[0]`, `remove`) — O(n) (нужен сдвиг элементов).
- Поиск по значению (`x in lst`, `lst.index(x)`, `lst.count(x)`) — O(n).

#### Что такое allocated?
Выделенная ёмкость списка: сколько слотов под указатели зарезервировано в памяти.

#### Что такое аллокация?
**Аллокация (allocation)** — выделение участка памяти под данные. От англ. _allocate_ — «выделять, отводить».

#### Что такое реаллокация?
Выделение нового, большего блока памяти. Иногда старый блок можно расширить на месте — если сразу за текущим блоком в куче есть свободное пространство, но чаще выделяется новый блок.

#### Какой запас памяти у пустого списка?
Нулевой. Запас появляется при первом `append`. CPython при росте с нуля выделяет сразу 4 слота.

#### Как узнать allocated?
- `(sys.getsizeof(lst) - sys.getsizeof([])) // 8` — через размер буфера (на 64-бит).
- `ctypes.PyListObject.from_address(id(lst)).allocated` — чтение поля C-структуры напрямую.

#### Что такое over-allocation?
Выделение большего объёма памяти, чем нужно прямо сейчас, чтобы создать запас на будущий рост и добавлять за O(1).

#### Почему амортизированная сложность append O(1), несмотря на периодические реаллокации?
Память выделяется с запасом, поэтому реаллокации редки и происходят всё реже, чем больше список.

#### Когда происходит реалокация списка?
В момент добавления нового элемента (`append`/`insert`/`extend`), если для него нет места (`ob_size == allocated`).

#### Что происходит при реалокации?
- Благодаря проверке (`len == allocated`) обнаруживается, что место кончилось.
- Выделяется новый, больший блок памяти под массив указателей (с запасом ~12.5%).
- Из старого блока в новый копируются указатели — просто адреса, по 8 байт каждый.
- Старый блок освобождается, список начинает ссылаться на новый.

#### На сколько увеличивается список при реалокации?
12.5% от newsize + 6 элементов, округлённое вниз до кратного 4:

```python
new_allocated = (newsize + (newsize >> 3) + 6) & ~3
```

#### Если мы в цикле добавляем элементы через append, как часто происходит реалокация?
0, 4, 8, 16, 25, 35, 46, 58, 72, 88, 106, 126, 148...
Чем больше список, тем реже реалокации: их количество растёт логарифмически, а не линейно.

#### Могут ли в списке Python быть «дыры» — пустые слоты между элементами?
Нет. Список всегда плотный — элементы лежат подряд, без пропусков. Разрежённости как явления в Python-списках не существует.

#### Что произойдёт, если у пустого списка выполнить lst[100] = 'x'?
`IndexError: list assignment index out of range`. Присваивать можно только по существующему индексу.
Расширять список можно только через `append`/`insert`/`extend`.

#### Что показывает sys.getsizeof()?
- `getsizeof` показывает размер объекта в байтах.
- `getsizeof(lst) = базовый + allocated × 8`.
- Базовый размер пустого списка: `getsizeof([]) = 56` байт (заголовок объекта + поля ob_size, allocated, указатель на буфер).
- Размер одного указателя в allocated — 8 байт.

#### Почему sys.getsizeof() для списка из 5 элементов может быть больше, чем для 4, но не изменится при добавлении 6-го?
Потому что `getsizeof` меряет выделенную ёмкость (allocated), а не len — а ёмкость растёт скачками при реалокациях, не на каждый элемент.

#### Освобождается ли зарезервированная память автоматически при удалении элементов из списка?
Да, если занято меньше половины ёмкости (`newsize < allocated/2`).
Выделяется блок поменьше, но не в притык, а с запасом ~12.5%.

#### Если количество элементов известно заранее, как избежать лишних реаллокаций?
List comprehension вместо цикла с append:

```python
result = [f(i) for i in range(n)]
```

CPython при создании списка из comprehension/литерала часто может оценить размер и выделить память один раз.

Предзаполнение через умножение:

```python
lst = [None] * n
```

#### Основные методы списка
- `append(x)` — добавить элемент в конец.
- `insert(i, x)` — вставить элемент по индексу `i`, сдвинув остальные вправо.
- `extend(iterable)` — добавить в конец все элементы из другого итерируемого объекта.
- `pop(i=-1)` — удалить и вернуть элемент по индексу (по умолчанию последний); на пустом списке кидает `IndexError`.
- `remove(x)` — удалить первое вхождение значения `x`; если такого значения нет, кидает `ValueError`.
- `clear()` — удалить все элементы.
- `copy()` — вернуть неглубокую копию списка (`lst.copy() == lst`, но `lst.copy() is not lst`).

```python
lst = [3, 1, 2]
lst.append(4)      # [3, 1, 2, 4]
lst.insert(0, 99)  # [99, 3, 1, 2, 4]
lst.pop()           # 4, lst = [99, 3, 1, 2]
lst.pop(0)          # 99, lst = [3, 1, 2]
lst.remove(1)       # lst = [3, 2]
```

#### Методы поиска и сортировки списка
- `index(x)` — вернуть индекс первого вхождения `x`; если элемента нет, кидает `ValueError`.
- `count(x)` — сколько раз `x` встречается в списке.
- `sort()` — отсортировать список на месте (in-place), возвращает `None`.
- `reverse()` — развернуть список на месте, возвращает `None`.

```python
lst = [3, 1, 2, 1]
lst.count(1)   # 2
lst.index(2)   # 2
lst.sort()     # lst = [1, 1, 2, 3]
lst.reverse()  # lst = [3, 2, 1, 1]
```
