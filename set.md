#### Что такое set?
**Set** (множество) — изменяемая неупорядоченная коллекция уникальных хешируемых элементов, реализованная через хеш-таблицу.
- Дубликаты автоматически отбрасываются.
- Элементы не хранят порядок вставки, порядок обхода определяется хешами и может отличаться от порядка добавления.
- Элементы должны быть хешируемыми 
- Изменяемый: можно добавлять и удалять элементы после создания.

#### Что такое frozenset?
**Frozenset** — неизменяемая версия `set`. Раз неизменяем — хешируем, в отличие от обычного `set`, и может быть ключом словаря или элементом другого множества.

#### Как устроен set под капотом?
Как и [[Dict|словарь]], `set` — это хеш-таблица, но упрощённая: хранит только хеши и сами значения, без пары "ключ-значение".
- Есть разреженный массив слотов, куда бьёт хеш элемента (аналог `indices` у словаря).
- В слоте хранится сам элемент (точнее, запись с его хешем и значением), а не отдельная ссылка на второй массив, как в dict.
- Проверка на равенство при коллизии — через `==`, как и в словаре.

#### Алгоритмическая сложность операций set?
- Добавление (`add`), удаление (`discard`/`remove`), проверка на вхождение (`in`) — O(1) в среднем.
- В худшем случае (много коллизий, ресайз) — O(n).
- Проверка на вхождение в `set` асимптотически быстрее, чем в `list` (O(1) против O(n)), потому что не нужен последовательный перебор элементов — сразу вычисляется позиция по хешу.

#### Какой размер у пустого set?
`sys.getsizeof(set()) = 216` байт — заметно больше, чем у пустого `list` (56 байт) или `dict`, так как под хеш-таблицу сразу выделяется минимальный размер (8 слотов).

#### Когда происходит resize set?
При заполнении на 3/5 (60%) — Set использует запас меньше, чем [[Dict|словарь]], resize происходит раньше.

#### Как создать set?
```python
s = {1, 2, 3}          # литерал
s = set([1, 2, 2, 3])  # из итерируемого объекта, {1, 2, 3}
s = set()              # пустой; {} создаёт dict, а не set
empty = set()
```

#### Основные операции над множествами
- `s.add(x)` — добавить элемент; если уже есть, ничего не происходит.
- `s.discard(x)` — удалить элемент, если он есть; если нет — молча ничего не делает.
- `s.remove(x)` — удалить элемент; если его нет, кидает `KeyError`.
- `s.pop()` — удалить и вернуть произвольный элемент; на пустом множестве кидает `KeyError`.

#### Операции над множествами как в математике
- Объединение: `a | b` или `a.union(b)`.
- Пересечение: `a & b` или `a.intersection(b)`.
- Разность: `a - b` или `a.difference(b)`.
- Симметрическая разность (элементы, которые есть только в одном из множеств): `a ^ b` или `a.symmetric_difference(b)`.

```python
a = {1, 2, 3}
b = {2, 3, 4}
a | b   # {1, 2, 3, 4}
a & b   # {2, 3}
a - b   # {1}
a ^ b   # {1, 4}
```

#### Как проверить отношения между множествами?
- `a.issubset(b)` или `a <= b` — все элементы `a` есть в `b`.
- `a.issuperset(b)` или `a >= b` — все элементы `b` есть в `a`.
- `a.isdisjoint(b)` — у множеств нет общих элементов.

#### Для чего использовать set вместо list?
- Когда нужна быстрая проверка на вхождение (`in`) — O(1) вместо O(n).
- Когда нужно убрать дубликаты из коллекции: `list(set(lst))` (порядок элементов не гарантируется).
- Когда нужны операции над множествами (объединение, пересечение и т.д.).
