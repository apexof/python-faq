#### Что такое tuple?
Tuple (кортеж) — неизменяемая упорядоченная последовательность элементов.
Но если внутри лежит изменяемый объект — его содержимое менять можно.

#### Алгоритмическая сложность операций tuple?
- Доступ по индексу (`t[i]`, `len(t)`) — O(1).
- Поиск по значению (`x in t`, `t.index(x)`, `t.count(x)`) — O(n) (линейный перебор, не хеш-таблица).
- Итерация (`for x in t`), агрегаты (`min`, `max`, `sum`) — O(n).
- Конкатенация (`t1 + t2`), повтор (`t * k`), срез (`t[a:b]`) — O(длины результата), создаётся новый кортеж.
- `hash(t)`, сравнение (`t1 == t2`) — O(n).

#### Можем ли использовать tuple (0, 1, ['a']) в качестве ключа для словаря?
Нет: ключ словаря должен быть хешируемым целиком, а список внутри кортежа — изменяемый и нехешируемый объект.
Можно использовать как ключ, только если все элементы внутри тоже неизменяемы.

#### Отличие tuple от list?
- Изменяемость: tuple — нет, list — да.
- Хешируемость: tuple можно использовать как ключ словаря / множества (если внутри всё хешируемо), list — нельзя.
- Память: tuple компактнее (нет over-allocation, один блок памяти), list больше.
- Скорость: tuple чуть быстрее на создании и доступе.
- Структура в C: у tuple массив указателей встроен в объект (одна аллокация), у list — отдельно выделенный массив (две аллокации, перевыделяется при росте).
- Методы: у tuple только `count` и `index`, у list много (`append`, `pop`, `sort`, ...).
- Семантика: tuple — фиксированная разнородная запись (`user = ("Alex", 28, "RU")  # имя, возраст, страна`), list — однородная коллекция переменной длины (`names = ["Alex", "Maria", "Ivan"]`).

#### Почему при создании кортежа происходит одна аллокация памяти, а при создании списка — две?
Кортеж неизменяемый — его размер известен раз и навсегда при создании. Поэтому CPython кладёт заголовок объекта и массив указателей на элементы в один цельный блок памяти → одна аллокация.

У списка две аллокации:
- Структура списка — заголовок объекта: счётчик ссылок, тип, длина, поле allocated и указатель `ob_item`.
- Массив указателей выносится отдельно, чтобы он мог расти, не меняя адрес самого объекта.

#### Почему tuple занимает меньше памяти, чем list того же размера?
- Одна аллокация: заголовок и массив указателей лежат в одном блоке (у list — две: структура и отдельный массив).
- Нет over-allocation: выделяется ровно по размеру, list держит запас слотов под рост.
- Нет поля allocated и лишнего указателя `ob_item` → структура компактнее.

#### Почему tuple создаётся быстрее, чем list?
- Метаданные и массив указателей лежат в одной аллокации, у списка — в двух.
- Constant folding — кортеж-литерал из констант создаётся один раз при компиляции, список строится заново каждый раз.
- Нет over-allocation — кортежу не нужен запас памяти под рост, в отличие от списка.
- CPython переиспользует готовые оболочки маленьких кортежей.

#### Почему tuple быстрее, чем list, при доступе по индексу?
Потому что у кортежа массив указателей встроен в сам объект и лежит в той же аллокации, а у списка — это отдельный блок памяти, до которого нужно дополнительно перейти по указателю `ob_item`.
