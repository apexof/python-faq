#### Что такое return self?
`return self` в методах класса обычно используют, когда реализуют паттерн fluent interface / method chaining, т.е. чтобы можно было вызывать методы цепочкой: `obj.set_a().set_b().build()`.

Такой подход удобен, когда метод изменяет состояние объекта и должен вернуть этот же объект, чтобы можно было продолжить работу без создания нового экземпляра.

Второй частый случай — `__iter__`, где объект сам является [[Итератор|итератором]]:

```python
def __iter__(self):
    return self  # объект — сам итератор
```

#### Что нужно сделать для того, чтобы == сравнивало у этих классов значение self.a?
Нужно переопределить метод `__eq__`

```python
class A:
    def __eq__(self, other):
        return self.a == other.a
```

если `x.__eq__(y)` не реализован или возвращает `NotImplemented`, то Python пробует `y.__eq__(x)`.

#### Что такое дандер-методы (Magic Methods)?
Дандер-методы (от double underscore) — это специальные методы в Python, чьи имена начинаются и заканчиваются двумя подчёркиваниями, например `__init__`, `__eq__`, `__next__`, `__iter__`.
Питон вызывает их неявно при определённых операциях.
Они позволяют переопределять встроенное поведение объектов (инициализация, сравнение, арифметика, доступ по индексу, контекстный менеджер и т.д.).

#### Три принципа ООП и в чем их суть?
- **Инкапсуляция** — прячем атрибуты, методы от внешнего мира.
    - **private** — пользователь не может напрямую изменить метод начинающийся с двух `__`. Интерпретатор переименовывает `__name` в `_ClassName__name` (name mangling). Главная цель — **не сокрытие, а избегание коллизий имён при наследовании**. Каждый класс получает «свой» namespace для таких атрибутов, и наследник случайно не перезатрёт внутренность родителя.
    - **protected** — имена начинающиеся с одного `_` аналог protected имен. Это просто соглашение, доступ к ним можно получить как обычно, но они не импортируются при `from module import *` (если не указаны в `__all__`).
- **Наследование** — создаём класс на основе другого, повторно используя код.
- **Полиморфизм** («много форм») — это когда один и тот же код работает с объектами разных типов, а конкретное поведение зависит от того, какой объект пришёл.

#### MRO
MRO (Method Resolution Order) — порядок, в котором Python ищет метод/атрибут по цепочке наследования.
Использует алгоритм **C3-линеаризация**:
- Сначала сам класс, потом родители слева направо
- Подкласс всегда раньше своих родителей
- Порядок родителей в объявлении сохраняется
- Если согласованного порядка нет — Python кидает `TypeError` при объявлении класса
**Как посмотреть:** `ClassName.__mro__` или `ClassName.mro()`
##### Пример 1:
Если ветки **не сходятся** в общем предке, MRO идёт в глубину по первой ветке до конца, потом по второй.
```python
class A: pass
class B: pass
class C(A): pass
class D(B): pass
class E(C, D): pass

E.__mro__
# E -> C -> A -> D -> B -> object
```
##### Пример 2:
Ромбовидное наследование
```python
class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass

D.__mro__
# D -> B -> C -> A -> object
```

`A` идёт после `C`, хотя `B` наследует `A` напрямую. C3 «откладывает» общего предка до конца, чтобы не нарушить порядок.
