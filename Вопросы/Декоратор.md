#### Что такое декоратор?
Функция, которая принимает другую функцию (или класс) и возвращает новую, добавляя поведение без изменения исходного кода.

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("До вызова функции")
        result = func(*args, **kwargs)
        print("После вызова функции")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Привет, {name}!")

say_hello("Алекс")
# До вызова функции
# Привет, Алекс!
# После вызова функции
```

`@my_decorator` — синтаксический сахар для `say_hello = my_decorator(say_hello)`.

#### Как вызвать оригинальную функцию, если к ней уже применён декоратор?
Достать оригинал через `__wrapped__`, если декоратор использует `functools.wraps`:

```python
import functools

def my_decorator(func):
    @functools.wraps(func)   # сохраняет __name__, __doc__ и кладёт оригинал в __wrapped__
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Привет, {name}!")

say_hello.__wrapped__("Алекс")  # вызов оригинала напрямую
```


Какими декораторами пользовался?
@staticmethod / @classmethod — фабричные методы.
@functools.lru_cache / @cache — мемоизация тяжёлых детерминированных вызовов: токенизация, загрузка эмбеддингов, препроцессинг.
@functools.wraps — обязательно внутри своих декораторов, чтобы не терять метаданные.

Что такое декоратор класса?
Декоратор класса — это функция, которая принимает класс как аргумент и может изменить, расширить или заменить его перед созданием экземпляров. В Python он применяется аналогично декораторам функций, только к классу.

def add_greeting(cls):
    cls.greet = lambda self: f"Привет, я {self.name}"
    return cls

@add_greeting
class Person:
    def __init__(self, name):
        self.name = name

p = Person("Алекс")
print(p.greet())   # Привет, я Алекс
@add_greeting = Person = add_greeting(Person). Декоратор добавил классу метод greet и вернул тот же класс.

Что такое миксины?
Миксины (mixins) — это вспомогательные классы, которые добавляют отдельную функциональность другим классам через множественное наследование, но не предназначены для самостоятельного использования. 

Зачем нужны @classmethod и @staticmethod?
По синтаксису это декораторы но по сути дескрипторы, они не оборачивают вызов, как пользовательский декоратор, а меняют то, как метод привязывается к объекту при доступе через протокол дескрипторов (__get__).

@classmethod
Первым параметром получает ссылку на класс, а на экземпляр.

Зачем нужен:
Альтернативные конструкторы (factory methods)
Работа с атрибутами класса
Правильно работает с наследованием, cls всегда указывает на тот класс, через который вызвали метод.


Примеры использования @classmethod
datetime.fromtimestamp(), datetime.now() — фабрики datetime.
dict.fromkeys().

@staticmethod
Не получает ни self, ни cls - обычная функция внутри класса

Зачем нужен:
Утилитарные функции, логически связанные с классом
Группировка функций в namespace класса
Не нужен доступ к данным класса/экземпляра
