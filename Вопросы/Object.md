Как устроена память в питоне?

1. Все - объекты
Переменные хранят ссылки на объекты, а не сами значения
Объект содержит: значение, тип, счетчик ссылок
a = b не копирует объект, а добавляет ещё одну ссылку на тот же объект.

a = [1, 2, 3]  # a ссылается на список
b = a          # b ссылается на тот же список


2. Reference Counting
Каждый объект имеет счетчик ссылок ob_refcnt
Когда счетчик = 0, память освобождается

import sys
sys.getrefcount(a)  # показывает количество ссылок

3. Garbage Collector
Дополнительно к reference counting
Находит и удаляет циклические ссылки (A → B → A)
Работает периодически или при превышении порогов

Что такое garbage collector? Что такое цикличные ссылки?
Garbage collector (GC) — автоматическая система удаления объектов, на которые больше никто не ссылается.
Циклические ссылки — объекты, которые ссылаются друг на друга:
a = []
b = []
a.append(b)
b.append(a)
​
Они "держат" друг друга, хотя больше никому не нужны. Python умеет находить такие циклы и удалять.

В чем разница между == и is?
= - сравнение значений
Проверяет равны ли значения объектов
Вызывает метод __eq__()
is - сравнение идентичности
Проверяет являются ли две переменные одним и тем же объектом в памяти
Сравнивает id() объектов
# Списки с одинаковым содержимым
a = [1, 2, 3]
b = [1, 2, 3]

a == b  # True - значения равны
a is b  # False - разные объекты в памяти

# Одна ссылка
c = a
c is a  # True - один объект
c == a  # True - значения тоже равны



Почему 
a = 256
b = 256
a is b   # True  — один закэшированный объект

a = 257
b = 257
a is b   # False — два разных объекта

Это interning (кэширование) мелких целых чисел в CPython.
При старте интерпретатор создаёт пул объектов int для диапазона от -5 до 256 включительно. Эти объекты существуют в единственном экземпляре, поэтому любая переменная с таким значением ссылается на один и тот же объект.

Кэшируются ли строки так же?
Да, есть string interning, но правила другие: интернируются строки, похожие на идентификаторы (буквы/цифры/_). Можно форсировать через sys.intern().
