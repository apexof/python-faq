**Итератор** — объект, по которому можно проходить по одному элементу.

```python
it = iter([1, 2, 3])
print(next(it))  # 1
```

**Протокол итерации** требует двух методов:

- `__iter__()`: Возвращает сам объект-итератор.
- `__next__()`: Возвращает следующее значение или выбрасывает `StopIteration`

Пример итератора (счетчик до `n`):

```python
class Counter:
    def __init__(self, low, high):
        self.current = low
        self.high = high

    def __iter__(self):
        return self  # Итератор возвращает сам себя

    def __next__(self):
        if self.current > self.high:
            raise StopIteration
        else:
            self.current += 1
            return self.current - 1
```
Зачем нужен?
- **Дать единый способ перебора** — `for` работает одинаково с list, dict, set, файлом и любым своим объектом.
- **Перебирать лениво** — по одному элементу, не загружая всё в память.
- **Скрыть внутреннее устройство** — потребитель просто берёт `next()`, не зная, как объект хранит данные.

Что такое Генератор?
**Генератор** — функция с `yield`, которая возвращает итератор, производящий значения лениво (по одному, по запросу), а не сразу весь список.
**Генератор** — это способ быстро создать итератор с `yield`:

```python
def my_gen():
    yield 1
    yield 2

for x in my_gen():
    print(x)
```

Зачем нужен?
- **Экономить память** — хранят только текущее состояние, а не все элементы.
- **Работать с большими/бесконечными потоками** — данные не помещающиеся в память.
- **Считать лениво** — значения вычисляются только когда нужны.

Как создать генератор?
**Функция-генератор** — `def` с `yield` внутри:

```python
def gen():
    yield 1
    yield 2
```

**2. Generator expression** — генераторное выражение, вообще без функции и без `yield`:

```python
g = (x*x for x in range(10))   # это уже генератор
type(g)                         # <class 'generator'>
```

Что такое iterable?
**Iterable** — это объект, который можно перебрать в цикле `for`. По сути — то, у чего есть метод `__iter__()`, способный отдать итератор.
Примеры: list, str, dict, set, range, файл.

Отличие iterable от итератора?
- **Iterable** — объект, который можно перебирать. У него есть метод `__iter__()`, возвращающий итератор. Сам элементы по одному не выдаёт. Примеры: list, str, dict, set, range.
- **Итератор** — объект, который реально ведёт перебор. У него есть `__next__()` (выдаёт следующий элемент) и `__iter__()` (возвращает себя же). Получается из iterable через `iter()`.
- **Состояние** — iterable не помнит, где ты остановился; итератор помнит позицию и при каждом `next()` продвигается дальше, пока не кончатся элементы (тогда `StopIteration`).
- **Проходы** — iterable перебирается многократно (каждый `for` берёт у него свежий итератор); итератор одноразовый — дойдя до конца, остаётся пустым, заново не начнёт.
- **Связь** — `iter(iterable)` возвращает итератор; цикл `for` делает это сам: сначала `iter()`, потом `next()` в цикле до `StopIteration`.
