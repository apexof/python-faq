#### Что такое переменная?
Переменная - это ссылка на объект в памяти

#### Что происходит когда мы присваиваем одну переменную в другую?
Когда вы копируете переменную, вы копируете ссылку, а не сам объект.

#### Глобальные и локальные переменные
Локальная переменная — создаётся внутри функции и видна только там.
Глобальная переменная — создаётся вне функции и доступна везде.
Чтобы изменить глобальную переменную в функции, надо писать `global x`.
Чтобы изменить локальную переменную в объемлющей функции, надо писать `nonlocal x`

#### Почему не рекомендуется изменяемый тип как значение по умолчанию?
Данные изменяются и эти изменения сохраняются между вызовами. Если попытаться изменить неизменяемый объект то будет создан новый объект и локальная переменная будет указывать на него.

```python
# Неправильно:
def add_item(item, target_list=[]): # Опасно!
    target_list.append(item)
    return target_list

# Правильно:
def add_item(item, target_list=None):
    if target_list is None:
        target_list = []
    target_list.append(item)
    return target_list
```

#### Чем отличается параметр от аргумента?
- **Параметр** — имя в определении функции (`def f(x):` — `x` параметр).
- **Аргумент** — конкретное значение, которое передают при вызове (`f(5)` — `5` аргумент).

#### Какие бывают виды аргументов?
- **Позиционные (positional)** — передаются по порядку: `f(1, 2)`.
- **Именованные (keyword)** — передаются по имени: `f(a=1, b=2)`. Порядок не важен.

При **вызове** позиционные аргументы должны идти до именованных, иначе `SyntaxError`:

```python
f(1, b=2)   # ок
f(a=1, 2)   # SyntaxError
```

#### Какие бывают виды параметров?
- **Обязательные** — без значения по умолчанию: `def f(a):`.
- **Со значением по умолчанию** — `def f(a, b=10):`. Должны идти после обязательных.
- **`*args`** — собирает лишние позиционные аргументы в кортеж.
- **`**kwargs`** — собирает лишние именованные аргументы в словарь.

```python
def f(a, b=10, *args, **kwargs):
    return a, b, args, kwargs

f(1, 2, 3, 4, x=5)
# a=1, b=2, args=(3, 4), kwargs={'x': 5}
```

`args` и `kwargs` — просто соглашение, имена можно любые (`*items`, `**options`). Важны именно `*` и `**`.

#### Какой порядок параметров в определении функции?
Строго: сначала обязательные позиционные, затем со значением по умолчанию, затем `*args`, затем keyword-only, затем `**kwargs`.

```python
def f(a, b=10, *args, kw_only, **kwargs):
    ...
```

- `/` — всё, что слева от него, можно передать **только позиционно** (positional-only).
- `*` (или `*args`) — всё, что справа, можно передать **только по имени** (keyword-only).

```python
def f(pos_only, /, normal, *, kw_only):
    ...

f(1, 2, kw_only=3)        # ок
f(1, normal=2, kw_only=3) # ок
f(pos_only=1, ...)        # TypeError — pos_only нельзя по имени
```
