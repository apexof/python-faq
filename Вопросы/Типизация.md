#### Какие типы изменяемые а какие нет?
Изменяемые: list, dict, set, bytearray (еще класс, функция, модуль, инстанс класса)
Неизменяемые: int, float, str, tuple, frozenset

#### Какая типизация в Python?

Динамическая — тип переменной определяется во время выполнения:

```python
x = 5       # int
x = "text"  # str - переприсвоение без ошибки
```

Строгая — Python не выполняет неявных приведений между несовместимыми типами.

`"5" + 5` бросит `TypeError`, а не вернёт `"55"` или `10`, как в слабо типизированных языках (например, JavaScript).

#### Какие виды приведения типов есть в Python?

Приведение типов бывает двух видов: явное и неявное.

Неявное (implicit) — интерпретатор сам приводит типы в безопасных случаях, без потери данных. В основном это касается числовых операций:

```python
x = 5 + 2.0   # int + float -> float, x = 7.0
y = True + 1  # bool -> int, y = 2 (bool это подкласс int)
```

На несовместимых типах неявного приведения не будет (сильная типизация) — `"5" + 5` даст `TypeError`.

Явное (explicit, оно же type casting) — ты сам вызываешь функцию-конструктор типа:

```python
int("42")        # 42
int(3.9)         # 3 (отбрасывает дробную часть, не округляет)
float("3.14")    # 3.14
str(42)          # "42"
list("abc")      # ['a', 'b', 'c']
tuple([1, 2])    # (1, 2)
set([1, 1, 2])   # {1, 2}
dict([("a", 1)]) # {"a": 1}
bool(0)          # False
```

Несколько нюансов:
- `int()` от строки требует валидное целое — `int("3.14")` бросит `ValueError`, надо через `int(float("3.14"))`.
- Эти функции не «меняют» объект, а создают новый — объекты неизменяемых типов в Python не мутируют.
- `bool()` следует правилам truthiness: пустые коллекции, `0`, `0.0`, `""`, `None` → `False`, остальное → `True`.
- Для пользовательских классов поведение приведения задаётся дандер-методами: `__int__`, `__float__`, `__str__`, `__bool__`, `__index__` (для использования объекта как индекса/в `bin()`, `hex()` и т.п.).

#### Какие значения считаются ложными (falsy)?
Основные:


```python
None
False
0    # int
0.0  # float
""   # пустая строка
[]   # пустой список
()   # пустой кортеж
{}   # пустой словарь
set()  # пустое множество
frozenset()     # пустое неизменяемое множество
```

Редкие/экзотика:

```python
0j              # complex (комплексный ноль)
Decimal(0)      # из модуля decimal
Fraction(0, 1)  # из модуля fractions
range(0)        # пустой range
b""             # пустые bytes
bytearray(b"")  # пустой bytearray
```

Логика сводится к трём правилам:

- Числовой ноль любого типа — falsy.
- Любая пустая коллекция/последовательность — falsy.
- `None` и `False` — falsy.

Непустая коллекция всегда truthy, даже если внутри лежат фолси-значения

Под капотом truthiness определяется дандер-методами: сначала вызывается `__bool__()`, а если его нет — `__len__()` (объект ложный, если длина 0). Если нет ни того, ни другого — объект всегда truthy. Поэтому свои классы можно делать falsy:

```python
class Empty:
    def __bool__(self):
        return False
```
