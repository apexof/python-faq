Контекстный менеджер — это объект, который управляет ресурсом: гарантированно делает что-то «на входе» и что-то «на выходе» из блока `with`, даже если внутри произошла ошибка. 

Классический пример — открыть файл и точно его закрыть.
```python
with open("file.txt") as f:
    data = f.read()
# здесь файл уже закрыт — автоматически, даже если был exception
```

Объект становится контекстным менеджером, если у него есть два дандер-метода:
- `__enter__(self)` — выполняется при входе в `with`. То, что он вернёт, попадёт в переменную после `as`.
- `__exit__(self, exc_type, exc_val, exc_tb)` — выполняется при выходе из блока 
	- `exc_type` — класс исключения (или `None`, если ошибки не было)
	- `exc_val` — сам объект исключения (или `None`)
	- `exc_tb` — traceback (или `None`)
	- Вернёт `True` — исключение подавляется; `False`/`None` — пробрасывается дальше

### Что будет если внутри произойдет ошибка?
`__exit__` **всё равно вызовется**, и в его аргументы придёт информация об исключении. Ресурс корректно освободится. А дальше зависит от того, что вернёт `__exit__`:
Два сценария:
- `__exit__` вернул `False`/`None` — очистка произошла, потом исключение **летит дальше** наружу (поведение по умолчанию). Нужно обрабатывать исключение снаружи. 
- `__exit__` вернул `True` — очистка произошла, исключение **подавляется**, программа продолжает работать после блока `with`.
