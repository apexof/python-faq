#### Что такое view-объект?
**View-объект** — объект, который не хранит и не копирует данные, а предоставляет к ним динамический доступ на чтение из другой структуры. При изменении исходной структуры все ранее полученные view сразу отражают эти изменения — в этом и есть смысл: view не снимок на момент вызова, а постоянно актуальное «окно» в данные.

```python
d = {'a': 1, 'b': 2}
keys = d.keys()       # view-объект dict_keys
d['c'] = 3
print(keys)  # dict_keys(['a', 'b', 'c']) — новый ключ уже виден без повторного вызова
```

#### Что можно делать с view-объектом?
Через view можно итерироваться, узнавать длину и проверять вхождение, но не обращаться по индексу — обращение по индексу не имеет смысла, так как view не хранит элементы как последовательность:

```python
list(d.keys())   # ['a', 'b', 'c']
len(d.keys())    # 3
'a' in d.keys()  # True
d.keys()[0]      # TypeError: 'dict_keys' object is not subscriptable
```

#### Где в Python встречаются view-объекты?
- `dict.keys()`, `dict.values()`, `dict.items()` — [[Dict|словарь]] отдаёт `dict_keys`, `dict_values`, `dict_items`.
- `memoryview` — окно на буфер байтов другого объекта (`bytes`, `bytearray`, `array.array`) без копирования данных.

```python
import array
a = array.array('i', [1, 2, 3])
mv = memoryview(a)
a[0] = 99
print(mv.tolist())  # [99, 2, 3] — изменение исходного буфера видно через view
```

У `list`, `set`, `tuple` подобных «живых» view через методы нет: `list.copy()`, срез `lst[:]` и теоретико-множественные операции всегда возвращают независимую копию, не привязанную к изменениям оригинала.

#### Поддерживают ли view-объекты операции над множествами?
`dict.keys()` и `dict.items()` поддерживают теоретико-множественные операции (`&`, `|`, `-`, `^`), потому что ключи словаря уникальны и хешируемы — как элементы [[set]]. `dict.values()` их не поддерживает: значения не обязаны быть уникальными и хешируемыми.

```python
d = {'a': 1, 'b': 2, 'c': 3}
d2 = {'b': 2, 'c': 3, 'd': 4}
d.keys() & d2.keys()      # {'b', 'c'} — общие ключи
d.keys() | d2.keys()      # {'a', 'b', 'c', 'd'} — объединение
d.values() & d2.values()  # TypeError: unsupported operand type(s) for &
```
