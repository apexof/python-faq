#### Что выведет код?
```python
s = 'Hello world'
s[5] = 'a'
```
`TypeError: 'str' object does not support item assignment` — строки в Python неизменяемые.

#### Что выведет код?
```python
a = []
b = a
a = [1]
print(b)
```
`[]` — `b` ссылается на старый список, а `a` после переприсваивания указывает на новый.

#### Что выведет код?
```python
a = []
b = a
b.append(1)
print(a)
```
`[1]` — `a` и `b` указывают на один и тот же список, `append` меняет его на месте.

#### Когда пустой список уйдёт из памяти?
```python
a = []
b = a
a = 0
b = 0  # уйдёт тут
```
В этот момент на `[]` остаётся 0 ссылок — `a` и `b` теперь указывают на числа.

#### Что выведет код?
```python
x = [[0] for _ in range(3)]
x[0][0] = 1
print(x)
```
`[[1], [0], [0]]` — list comprehension создаёт три независимых списка `[0]`, меняется только первый элемент `x`.

#### Что выведет код?
```python
x = 5
def f():
    x = 10
f()
print(x)
```
`5` — внутри функции `x = 10` создаёт новую локальную переменную, она не влияет на глобальную `x`.

Чтобы изменить глобальную переменную, нужен `global`:
```python
x = 5
def f():
    global x
    x = 10
f()
print(x)
```
`10`.

#### Что выведет код?
```python
x = [[0]] * 3
x[0][0] = 1
print(x)
```
`[[1], [1], [1]]` — `[[0]] * 3` создаёт список из трёх ссылок на один и тот же внутренний список `[0]`, изменение через любую из них видно во всех.

#### Что выведет код?
```python
a = 1
b = 1
print(a is b)
```
`True` — малые целые числа (от -5 до 256) интернируются: CPython заранее создаёт их как singleton-объекты, и все переменные с таким значением ссылаются на один и тот же объект.

Если `a` и `b` — одинаковые строковые литералы, результат `is` также может быть `True`: CPython интернирует строковые литералы на этапе компиляции. Это деталь реализации CPython, а не гарантия языка, поэтому сравнивать объекты на равенство значений нужно через `==`, а не `is`.

#### Что выведет код?
```python
print(0.25 + 0.25 == 0.5)
print(0.1 + 0.2 == 0.3)
```
`True` и `False`.

Числа с плавающей точкой хранятся в двоичной системе по стандарту **IEEE 754**.
`0.25` — степень двойки (`2**-2`), такие числа представляются в `float` точно, поэтому операции с ними точны.
`0.1` и `0.3` в десятичной системе — конечные дроби, но в двоичной они представляются как бесконечные периодические дроби, поэтому при сложении накапливается погрешность округления. Для точных вычислений использовать `decimal.Decimal`.

```python
from decimal import Decimal
print(Decimal("0.1") + Decimal("0.2") == Decimal("0.3"))   # True
# важно: строки, а не Decimal(0.1) — иначе подхватишь ту же ошибку float
```

#### Почему `a is b` для 256, но не для 257?
```python
a = 256
b = 256
a is b   # True  — один закэшированный объект

a = 257
b = 257
a is b   # False — два разных объекта
```

Это interning (кэширование) мелких целых чисел в CPython.
При старте интерпретатор создаёт пул объектов int для диапазона от -5 до 256 включительно. Эти объекты существуют в единственном экземпляре, поэтому любая переменная с таким значением ссылается на один и тот же объект.

#### Что нужно сделать для того, чтобы == сравнивало у этих классов значение self.a?
Нужно переопределить метод `__eq__`

```python
class A:
    def __eq__(self, other):
        return self.a == other.a
```

если `x.__eq__(y)` не реализован или возвращает `NotImplemented`, то Python пробует `y.__eq__(x)`.
