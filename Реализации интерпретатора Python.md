**Реализация интерпретатора Python** — конкретная программа, которая исполняет код Python; спецификация языка не привязана к единственной реализации, поэтому существует несколько интерпретаторов с разными целями и внутренним устройством.

## CPython

**CPython** — эталонная (reference) реализация, написанная на C; именно её по умолчанию имеют в виду под словом "Python" и распространяют с python.org.

- Компилирует исходный код в байт-код (`.pyc`), который исполняет стековая виртуальная машина.
- Использует [[GIL]] (Global Interpreter Lock) — в один момент времени только один поток исполняет байт-код Python.
- Управление памятью — счётчик ссылок (reference counting) + generational garbage collector для циклических ссылок.
- Эталон совместимости: новые версии языка и PEP реализуются здесь первыми, остальные реализации догоняют.

## PyPy

**PyPy** — реализация с JIT-компиляцией (just-in-time): часто исполняемый байт-код компилируется в машинный код на лету.

- На длительных вычислительных нагрузках значительно быстрее CPython (за счёт JIT и трассировки горячих участков кода).
- Сам PyPy написан на RPython (ограниченном подмножестве Python), который транслируется в C.
- Имеет собственный GIL — многопоточность с тем же ограничением, что и в CPython.
- Минус: расширения на C (например, многие пакеты с native-биндингами) могут быть несовместимы или работать через слой эмуляции `cpyext` медленнее, чем в CPython.

## Jython

**Jython** — реализация, компилирующая код Python в байт-код JVM (Java Virtual Machine).

- Прямая интеграция с Java: можно импортировать и использовать Java-классы из Python-кода.
- Не имеет GIL — потоки управляются стандартными механизмами JVM.
- Поддерживает только Python 2 (разработка под Python 3 не завершена).

## IronPython

**IronPython** — реализация для .NET / Mono, аналог Jython, но для платформы Microsoft.

- Интеграция с .NET-библиотеками и сборками (assemblies).
- Также не имеет GIL — потоки работают через CLR (Common Language Runtime).
- Как и Jython, актуальная поддержка ограничена линейкой Python 2; полноценный Python 3 не реализован.

## Сравнение

| Реализация | Язык реализации | Целевая платформа | GIL | Компиляция | Поддержка Python 3 |
|---|---|---|---|---|---|
| CPython | C | Кросс-платформенная (нативный код) | Да | В байт-код + интерпретация | Да, эталонная |
| PyPy | RPython (→ C) | Кросс-платформенная | Да | JIT в машинный код | Да |
| Jython | Java | JVM | Нет | В байт-код JVM | Нет (только Python 2) |
| IronPython | C# | .NET / CLR | Нет | В байт-код CLR | Нет (только Python 2) |
