За какую асимптотику работает базовая сортировка в Python?
Дефолтная сортировка в Python (

sort()

и

sorted()

) реализована алгоритмом Timsort.

В среднем и худшем случае — O(n log n), но на частично отсортированных данных может быть O(n).

Какие знаешь алгоритмы сортировки?
- **Timsort** — то, что внутри `sorted()` / `list.sort()`. Гибрид merge + insertion. Стабильный, адаптивный, O(n log n) худший, O(n) на почти отсортированных. 
Ищет в массиве уже упорядоченные участки («runs»), короткие достраивает insertion'ом, затем сливает runs как в merge sort.
- **Quicksort** — среднее O(n log n), худший O(n²), in-place, кэш-эффективный.  
Выбирает опорный элемент (pivot), разбивает массив на «меньше pivot» и «больше pivot», рекурсивно сортирует обе части.
- **Merge sort** — стабильный, гарантированный O(n log n), память O(n).  
Делит массив пополам до одиночных элементов, затем попарно сливает обратно в отсортированном порядке.
- **Heapsort** — O(n log n), in-place, нестабильный, плохой по кэшу.  
Строит из массива кучу (heap), затем по одному извлекает максимум/минимум из вершины в конец массива.
- **Insertion sort** — O(n²), но быстр на маленьких/почти отсортированных (Timsort юзает для коротких runs).  
Идёт по массиву и каждый элемент вставляет на своё место среди уже отсортированных слева — как сортировка карт в руке.
- **Counting sort** — O(n+k), когда ключи — целые в ограниченном диапазоне.  
Считает, сколько раз встречается каждое значение, затем восстанавливает массив по этим счётчикам (без сравнений).
- **Radix sort** — O(n·d), для целых/строк фиксированной длины.  
Сортирует поразрядно (от младшего разряда к старшему), на каждом разряде используя стабильную counting sort.
### Разница между поверхностной и глубокой копией?
Поверхностная (copy.copy) — копирует верхний уровень, вложенные объекты остаются общими по ссылке:
По сути создается новый объект того же типа и в него помещаются копии ссылок.
```python
import copy
a = [[1, 2]]

b = copy.copy(a)
b[0].append(9)
print(a)  # [[1, 2, 9]] — задело оригинал
```

Глубокая (copy.deepcopy) —  рекурсивно создаёт новые объекты-дубликаты для всех изменяемых данных, а для неизменяемых переиспользуются те же ссылки.
```python
import copy
a = [[1, 2]]

b = copy.deepcopy(a)
b[0].append(9)
print(a)  # [[1, 2] — без изменений
```
