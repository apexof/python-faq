#### За какую асимптотику работает базовая сортировка в Python?
Дефолтная сортировка (`list.sort()` и `sorted()`) реализована алгоритмом Timsort.

В среднем и худшем случае — O(n log n), на частично отсортированных данных может быть O(n).

#### Какие знаешь алгоритмы сортировки?
- **Timsort** — Гибрид merge + insertion.
  **Асимптотика:** O(n log n) худший, O(n) на почти отсортированных.
  Ищет в массиве уже упорядоченные участки («runs»), короткие достраивает insertion'ом, затем сливает runs как в merge sort.
- **Quicksort** — in-place, кэш-эффективный.
  **Асимптотика:** среднее O(n log n), худший O(n²).
  Выбирает опорный элемент (pivot), разбивает массив на «меньше pivot» и «больше pivot», рекурсивно сортирует обе части.
- **Merge sort** — стабильный, требует доп. память O(n).
  **Асимптотика:** гарантированный O(n log n) по времени.
  Делит массив пополам до одиночных элементов, затем попарно сливает обратно в отсортированном порядке.
- **Heapsort** — in-place, нестабильный, плохой по кэшу.
  **Асимптотика:** O(n log n).
  Строит из массива кучу (heap), затем по одному извлекает максимум/минимум из вершины в конец массива.
- **Insertion sort** — быстр на маленьких/почти отсортированных
  **Асимптотика:** O(n²).
  Идёт по массиву и каждый элемент вставляет на своё место среди уже отсортированных слева — как сортировка карт в руке.

#### Разница между поверхностной и глубокой копией?
Поверхностная (`copy.copy`) — копирует верхний уровень, вложенные объекты остаются общими по ссылке. По сути создаётся новый объект того же типа, и в него помещаются копии ссылок.

```python
import copy
a = [[1, 2]]

b = copy.copy(a)
b[0].append(9)
print(a)  # [[1, 2, 9]] — задело оригинал
```

Глубокая (`copy.deepcopy`) — рекурсивно создаёт новые объекты-дубликаты для всех изменяемых данных, а для неизменяемых переиспользуются те же ссылки.

```python
import copy
a = [[1, 2]]

b = copy.deepcopy(a)
b[0].append(9)
print(a)  # [[1, 2]] — без изменений
```

#### Замыкание
Замыкание - это способность функции запомнить внешнее окружение в котором она была создана и получить доступ к переменным из этого окружения даже после того как  внешняя функция завершила выполнение.
В `func.__closure__` лежит кортеж со значениями  замкнутых переменных
