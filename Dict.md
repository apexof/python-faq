#### Словарь
Словарь (dict) - структура данных "ключ-значение", реализованая через хеш-таблицу.
Ключи прогоняются через hash - на выходе получаем целое число.
Хеши используются для быстрого доступа к данным словаря.
Словарь под капотом это 2 массива indices и entries.
#### Как именно осуществляется доступ к данным по хешу?
my_object = {"a": 1, "b": 2}
Хотим получить my_object["b"]

1. получаем хеш от ключа hash("b") = 6132913304770597809
2. от хэша берется остаток от деления на размер массива indices, в результате получаем индекс массива indices
3. обращаемся в массив indices по индексу 1: indices[1]. Получаем индекс массива entries
4. обращаемся по индексу к массиву entries: entries[1] - получаем ключ "b"
5. сравниваем ключ "b" по которому начинали поиск с ключом из entries через `==`.
6. если ключи равны возвращаем значение

**1. `indices`** — разреженный массив (с «дырками»). Именно сюда бьёт хеш, тут вычисляется индекс. Но хранит он не данные, а лишь номер — ссылку во второй массив.

**2. `entries`** — упорядоченный массив, дырки могут быть от удаления элементов. Тут лежат сами записи: `(хеш, ключ, значение)`, по порядку вставки.

```
indices (8 слотов):
[ _, 0, 1, _, _, _, _, _ ]
       ↑  ↑
   слот1 слот2

entries (компактно, по порядку вставки):
[0] (hash=12345, "a", 1)
[1] (hash=67890, "b", 2)
```

#### Почему в словаре ключом может быть только неизменяемый тип?
Потому что от ключей берется хеш. Чтобы хеш был стабилен между вызовами данные от которых берем хеш должны быть не изменяемыми.
Более того встроенные изменяемые типы (list, dict, set) специально сделали нехешируемыми.

#### Может ли быть ключом словаря изменяемый тип данных?
Да, может. Ключом словаря может быть любой хешируемый тип.

#### Какие типы хешируемы и при этом изменяемы?
- экземпляры класса
- функции, методы, `lambda`
- модули
- классы

Для этих типов hash берется от id. А так как адрес объекта в памяти никогда не меняется то и хеш стабилен.
Если переопределить метод `__hash__` так, чтобы он зависел от изменяемого состояния, то после мутации этого состояния мы потеряем доступ к данным по ключу.

```python
class Box:
    def __init__(self, n):
        self.n = n
    def __eq__(self, other):
        return isinstance(other, Box) and self.n == other.n
    def __hash__(self):
        return hash(self.n)

key = Box(1)
d = {key: "value"} # экземпляр класса как ключ словаря
print(d[key])      # "value" — ок
key.n = 2          # мутируем поле, от которого зависит хеш
print(d[key])      # KeyError — хеш теперь hash(2), ищем не в том bucket
```

#### Для каких типов хеш берется по значению, а для каких по id?
Неизменяемые хешируются по значению
Изменяемые хешируются по id
Встроенные изменяемые типы (list, dict, set) специально сделали нехешируемыми.

```python
list.__hash__ is None        # True
```

#### Операции и алгоритмическая сложность
Операции: доступ, вставка, удаление — O(1) в среднем.
В худшем случае O(n) при вставке/удалении с ресайзом.

#### Коллизия
Коллизия — это когда два разных ключа дают один и тот же хеш. 
Происходят чаще обычного потому что хеши усекаются до размера таблицы (hash % table_size).
#### Способы разрешения коллизий
**1. Открытая адресация** — ищем другой свободный слот по определённому правилу (пробирование/probing). Так работает в Python.
**2. Метод цепочек** — в слоте лежит связный список, все ключи-коллизии висят в нём.

#### Когда происходит resize словаря?
При достижении 2/3 заполненности. Увеличивается на 3* используемый размер и округляется до степени 2.
Например было `6/8 -> 6*3 = 18` -> округляем до степени 2 -> 32 -> новый размер 6/32
При ресайзе увеличиваются оба массива (indices, entries). Если в entries были дырки они убираются.

#### Как записать и получить значение по ключу через []?
`d[key] = value` — если ключа нет, добавляет новую пару; если ключ уже есть, перезаписывает его значение. Одна и та же операция для обоих случаев, без разделения на "вставить" и "обновить".
`d[key]` — вернуть значение по ключу; если ключа нет, кидает `KeyError`.

```python
d = {'a': 1}
d['a'] = 2   # ключ есть — перезапись, d = {'a': 2}
d['b'] = 99  # ключа нет — вставка, d = {'a': 2, 'b': 99}
d['z']       # KeyError: 'z'
```

#### Основные методы словаря
Методы нужны там, где `[]` не подходит: `[]` либо падает с `KeyError` на отсутствующем ключе, либо ничего не говорит о том, был ключ уже вставлен или нет.
- `get(key, default=None)` — вернуть значение по ключу без риска `KeyError`; если ключа нет, вернуть `default`.
- `setdefault(key, default=None)` — вернуть значение по ключу, а если ключа нет — одновременно вставить его со значением `default` и вернуть его. `d[key] = d.get(key, default)` потребовало бы двух обращений к словарю, `setdefault` делает это за одно.
- `update(other)` — добавить/перезаписать сразу несколько пар "ключ-значение" из другого словаря, итерируемого объекта пар или именованных аргументов, не выписывая `d[k] = v` для каждой пары по отдельности.
- `pop(key, default)` — удалить ключ и вернуть его значение; если ключа нет и `default` не передан, кидает `KeyError`. В отличие от `del d[key]`, отдаёт удалённое значение.
- `popitem()` — удалить и вернуть последнюю добавленную пару `(ключ, значение)` (LIFO-порядок); на пустом словаре кидает `KeyError`.
- `clear()` — удалить все элементы.
- `copy()` — вернуть неглубокую копию словаря.

```python
d = {'a': 1, 'b': 2}
d.get('z', 'default')     # 'default', вместо KeyError
d.setdefault('c', 3)      # 3, d = {'a': 1, 'b': 2, 'c': 3}
d.setdefault('a', 100)    # 1 — ключ уже был, значение не тронуто
d.update({'d': 4}, e=5)   # d = {'a': 1, 'b': 2, 'c': 3, 'd': 4, 'e': 5}
d.pop('a')                # 1
d.popitem()                # ('e', 5) — последняя добавленная пара
```

#### Что возвращают keys(), values() и items()?
- `keys()` — [[View-объект|view-объект]] `dict_keys` с ключами словаря.
- `values()` — [[View-объект|view-объект]] `dict_values` со значениями словаря.
- `items()` — [[View-объект|view-объект]] `dict_items` с парами `(ключ, значение)`.

```python
d = {'a': 1, 'b': 2}
d.keys()    # dict_keys(['a', 'b'])
d.values()  # dict_values([1, 2])
d.items()   # dict_items([('a', 1), ('b', 2)])
```

#### Как создать словарь с одинаковым значением для набора ключей?
`dict.fromkeys(iterable, value)` — classmethod, создаёт новый словарь с ключами из `iterable`, у каждого значение `value`.

```python
dict.fromkeys(['x', 'y'], 0)  # {'x': 0, 'y': 0}
```
