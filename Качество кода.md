#### Что такое type hints?
Type hints — синтаксис для указания ожидаемых типов переменных, аргументов и возвращаемых значений. На выполнение не влияют, используются статическими анализаторами (mypy, pyright) и IDE.

```python
from typing import List, Dict, Optional, Union  # 3.5+ типы надо импортировать

def process_data(items: List[str], limit: int = 10) -> Dict[str, int]:
    return {item: len(item) for item in items[:limit]}
```

```python
# 3.9+ generic-типы можно писать с маленькой буквы (PEP 585)
def process(items: list[str], flag: bool) -> dict[str, int]:
    return {}
```

Что дают:
- автодополнение в IDE
- раннее обнаружение ошибок
- самодокументирование кода

#### Что такое mypy?
Статический анализатор типов. Если в коде есть аннотации, mypy находит места, где передан неверный тип, ещё до запуска программы.

#### Что такое Pydantic ?
Валидация данных во время выполнения и автоматическое преобразование типов. 
Даёт:
- автоматическую валидацию
- преобразование типов
- понятные сообщения об ошибках
- подсказки в IDE
- автодокументацию (OpenAPI в FastAPI)

Нужен при работе с внешними данными: API, БД, конфиги, файлы.

#### Что такое линтеры?
Линтер (linter) — инструмент статического анализа кода. Он:
- находит ошибки до выполнения программы
- проверяет стиль кода
- выявляет потенциальные баги
- обеспечивает единообразие кода в команде

Конкретные инструменты:
- **Flake8** — объединяет PyFlakes (логические ошибки), pycodestyle (PEP 8) и McCabe (цикломатическая сложность).
- **Black** — автоформаттер: не проверяет, а сам исправляет форматирование кода.
- **Ruff** — линтер и форматтер. Быстрый. Объединяет Flake8 (с плагинами), isort, pydocstyle, pyupgrade, autoflake и частично Black.
- **isort** — сортирует и группирует импорты по PEP 8: стандартная библиотека, сторонние библиотеки, локальные модули.
